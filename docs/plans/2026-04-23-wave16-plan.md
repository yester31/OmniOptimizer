# Wave 16: post-T1 fact-driven scope

**Status**: DRAFT 2026-04-23. Waiting on PR #7 (T1 audit_capabilities) + PR #8 (T7 cache key fix) to land before starting any code work.

**Headline**: T1 results gut the original Wave 16 scope. **Three of four planned workstreams are now dead or deferred.** Remaining scope is a single ~50-LOC observability change plus a deprioritized Wave 17 carry-over.

## T1 facts driving this plan

Source: `results/_capabilities.json` (generated by `scripts/audit_capabilities.py`, PR #7).

| Fact | Impact on planned Wave 16 scope |
|------|--------------------------------|
| `has_mha_pattern: false` — 4 MatMul nodes but 0 feed Softmax directly | `disable_mha_qdq` schema field → **DEAD**. No MHA to protect. |
| `has_avx512_bf16: false`, `has_amx_tile: false` on i7-11375H | `#43_openvino_bf16` → **hardware-blocked on this host, DEFERRED to Wave 17+** |

| ORT 1.22 `ALL` over `EXTENDED` = NCHWc + MLAS layout rewrites | On Tiger Lake (no NCHWc kernels for all ops) the ALL-level passes run but may not pay off. **CANDIDATE for measurement**, not an automatic switch. |

## What's OUT of Wave 16 (explicitly killed / deferred)

- **`disable_mha_qdq` schema field.** Earlier agent research claim ("MHA present in YOLO26n") was factually contradicted by T1. Field would be dead weight.
- **`#43_openvino_bf16` recipe.** Hardware-blocked on the only current measurement host. Re-evaluate in Wave 17+ if hardware upgrades. Keeps spec in TODOS T3.
- **T2 `_node_patterns.py` DRY extraction.** Was gated on NNCF IgnoredScope work which is no longer Wave 16 scope. Re-gate on Wave 17+ if scope returns. Keeps spec in TODOS T2.
- **Bulk modelopt exclusion expansion** (MatMul blacklisting, attention layer exemption, etc.). No attention path exists. No evidence MatMul hurts accuracy.

## What's IN of Wave 16

### D1 — `Result.build_ceiling_breached` bool field (MLPerf round-trip) — TODOS T6

**Goal**: Wave 15 D3 added `MeasurementSpec.build_ceiling_s`, but the ceiling-exceeded signal lives only in stderr. Downstream consumers (`recommend.py`, CI, any future dashboard) can't detect a recipe hit the diagnostic ceiling.

Fix: schema field + round-trip.

- D1.1 `scripts/_schemas.py::Result` — add `build_ceiling_breached: Optional[bool] = None` (nullable for backward-compat with historical JSONs)
- D1.2 `scripts/run_trt.py::_build_engine` — set `build_ceiling_breached = build_time_s > _ceiling` when ceiling is set, pass through to `Result`
- D1.3 `scripts/recommend.py` — include a column or flag in the ranking table when any recipe has `build_ceiling_breached: true`
- D1.4 test — schema round-trip + breach detection

**Size**: ~40 LOC code + ~30 LOC test.
**Depends on**: PR #6 (Wave 15 D3 schema) merged first.
**Risk**: minimal — additive field, default None.

### D2 (optional) — Audit CPU recipes' `graph_optimization_level` — deferred decision

**Goal**: T1 notes `ORT_ENABLE_ALL` (level 99) adds NCHWc layout + MLAS fusion beyond `EXTENDED` (level 2). On Tiger Lake without matching NCHWc kernels, those passes waste build time.

Action: **measurement-only scout**, not a refactor.

- D2.1 Pick two representative CPU recipes (e.g. `#30_ort_cpu_fp32`, `#34_openvino_fp32` — the latter is OV not ORT, skip if ORT-only)
- D2.2 Run each at `ORT_ENABLE_EXTENDED` vs current `ORT_ENABLE_ALL`. Compare fps + build time.
- D2.3 If EXTENDED within ±2% fps AND ≥20% faster build → **FOLLOW-UP** (separate PR, flip level, measure all CPU recipes). If ALL wins or ties → **NO ACTION**, add an inline comment referencing T1 finding.

**Size**: 0 LOC code (measurement only). 1 doc `docs/improvements/2026-04-XX-wave16-ort-level-audit.md`.
**Depends on**: nothing blocking.
**Risk**: waste 30 min of measurement for a null result. Acceptable.

## Lessons for future planning

1. **Agent research claims need T1-style fact snapshots before being used as plan input.** The "MHA present" claim was never verified; it would have driven a schema field + test battery that was pure waste.
2. **Negative audit results are load-bearing.** T1 saved an estimated 200+ LOC of pointless Wave 16 code (MHA schema + IgnoredScope patterns + BF16 recipe + tests).
3. **Plans should ship only after their facts are frozen.** Attempting to plan Wave 16 at Wave 15 ship time would have over-scoped; T1 came in between and collapsed the scope.
